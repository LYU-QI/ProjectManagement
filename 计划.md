# ProjectLVQI 多租户改造开发计划

**基准版本：** `e3175ce`（与远程 `origin/master` 同步）
**制定日期：** 2026-03-22
**目标：** 将 ProjectLVQI 从单组织单体架构升级为多租户企业级研发管理平台

---

## 总体架构目标

```
super_admin（平台级）
  └─ OrgMember.orgRole（组织级：owner / admin / member / viewer）
       └─ ProjectMembership.role（项目级：director / manager / member / viewer）
```

### 任务 1：数据库 Schema 改造（1 周）

**目标：** 建立 Organization 层，为所有后续改造的基础

#### 1.1 新增模型（schema.prisma）

```prisma
enum OrgRole {
  owner
  admin
  member
  viewer
}

model Organization {
  id         String      @id @default(cuid())
  slug       String      @unique
  name       String
  plan       Plan        @default(FREE)
  maxMembers Int         @default(25)
  createdAt  DateTime    @default(now())
  updatedAt  DateTime    @updatedAt
  members    OrgMember[]
  projects   Project[]
  configs    Config[]
}

model OrgMember {
  id             String       @id @default(cuid())
  userId         Int
  organizationId String
  orgRole        OrgRole      @default(member)
  departmentId    String?
  createdAt      DateTime     @default(now())
  updatedAt      DateTime     @updatedAt
  user           User         @relation(fields: [userId], references: [id])
  organization   Organization @relation(fields: [organizationId], references: [id])

  @@unique([userId, organizationId])
  @@index([organizationId])
}
```

#### 1.2 Config 数据库模型（替代 .env 全局配置）

```prisma
model Config {
  id             String        @id @default(cuid())
  organizationId String?
  organization   Organization? @relation(fields: [organizationId], references: [id])
  key            String
  value          String
  createdAt      DateTime      @default(now())
  updatedAt      DateTime      @updatedAt

  @@unique([organizationId, key])
  @@index([organizationId])
}
```

#### 1.3 User 表追加字段

```prisma
model User {
  // ... 现有字段
  feishuOpenId  String?
  feishuUnionId String?
  defaultOrgId  String?
}
```

#### 1.4 所有现有表追加 organizationId

| 表 | 操作方式 | 说明 |
|----|---------|------|
| Project | 追加 `organizationId String` FK | 通过 Project 级联到子表 |
| ProjectMembership | 追加 `organizationId String` FK | 加速查询，避免 JOIN |
| AuditLog | 追加 `organizationId String?` | null 代表全局审计 |
| FeishuUser | 追加 `organizationId String?` | null 代表全局用户 |
| RiskRule | 追加 `organizationId String?` | null 代表全局规则 |
| RiskAlert | 追加 `organizationId String?` | 通过 RiskRule 级联 |
| RiskRuleLog | 追加 `organizationId String?` | 通过 RiskRule 级联 |
| FeishuDependency | 追加 `organizationId String?` | 全局依赖 |

## 开发原则

### 数据库迁移

- 必须使用 `--create-only` 审查 SQL 后再执行
- 禁止直接 `migrate deploy`
- 先在 staging 环境验证，编写回滚脚本
- 存量数据通过 seed 脚本归入 `default` org

### 安全

- 所有 API 端点必须经过 OrgGuard 校验
- Prisma 中间件作为数据隔离的最后防线
- super_admin 角色的 `bypassOrgFilter` 标记需严格控制使用范围
- 上线前必须完成 E2E 跨租户数据隔离测试

### 前端重构

- App.tsx 当前 2548 行，必须拆分
- 所有视图状态迁移到 Zustand store
- App.tsx 目标缩减至约 300 行
- 组织切换时清空所有 project 级缓存状态

### 代码规范

- 每个任务完成后代码可正常运行，不破坏已有功能
- 每个任务完成后运行 `npm run build` 验证无编译错误
- 每个任务完成后运行 `npm run -w backend test`（如有测试）
- 所有新增 API 必须有对应的前端调用或明确说明为后续铺垫
