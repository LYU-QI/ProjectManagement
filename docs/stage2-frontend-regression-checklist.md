# 阶段 2 前端回归清单

## 目标
- 确认任务中心相关恢复链路在前端可达、可理解、不会残留旧上下文。

## 自动化 Smoke
- `backend/test/e2e/stage2-navigation.spec.ts`
  - 登录成功
  - 可进入任务中心
  - 可进入飞书集成
  - 可进入 PM 助手
  - 可进入自动化规则
  - 可从任务中心失败项跳转到飞书集成，并显示恢复上下文
  - 可从任务中心失败项跳转到 PM 助手，并显示恢复上下文
  - 可从任务中心失败项跳转到自动化规则，并显示恢复上下文与失败日志

## 人工回归
1. 在任务中心点击失败聚合卡的 `前往处理`
   - 应跳到对应业务页
   - 页面顶部应显示“来自任务中心的恢复上下文”
2. 飞书集成页
   - 若为组织级飞书表，应自动带入项目筛选
   - 应自动触发一次列表刷新
3. PM 助手页
   - 应自动展开日志筛选
   - 应自动切到 `失败` 日志视角
4. 自动化规则页
   - 应自动切到 `失败日志` 视角
   - 应自动展开一条存在失败日志的规则
5. 恢复上下文消费
   - 从任务中心跳过去后，再切换到其他页面，不应继续残留旧恢复上下文
   - 当前页顶部提示可以手动关闭

## 建议执行命令
- `npm run -w frontend build`
- `npm run -w backend build`
- `npm run -w backend test:e2e -- --grep "Stage 2 Navigation Smoke"`
