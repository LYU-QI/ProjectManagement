# ProjectLVQI 发布说明模板

本文档用于每次试运行、预发布或正式发布时记录本次版本的关键信息。目标不是写成长篇 changelog，而是让业务、实施、测试、运维都能快速知道：

- 这次发布了什么
- 会影响谁
- 需要注意什么
- 出问题怎么回退

## 1. 版本信息

- 版本号：
- 发布日期：
- 发布环境：`staging` / `production`
- 发布负责人：
- 审批人：

- 前端：

- 数据库迁移：

- 外部依赖变更：

## 3. 本次重点改动

- 改动 1：
- 改动 2：
- 改动 3：

建议写法：

- 面向业务结果，而不是仅写技术名词
- 同时标明影响模块，例如 `飞书集成`、`PM 助手`、`自动化规则`

## 4. 风险提示

- 飞书/AI/自动化外部依赖可用
- 回滚包或上一个稳定版本可用
