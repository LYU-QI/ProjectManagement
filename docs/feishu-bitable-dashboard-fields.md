# 飞书多维表格大看板字段清单

本文档按当前代码实现梳理所有与飞书多维表格打通的大看板、维护台和飞书记录列表字段。

说明：

- “字段名 / 支持别名”来自当前代码字段映射；配置中可通过字段映射 JSON 覆盖。
- “字段类型”是系统读取和维护时的建议类型，不等同于飞书侧强校验。
- 多数字段最终会被系统归一化为文本读取，因此文本、单选、人员等字段只要能被转成可读文本，通常都能显示。
- “用途”分为展示、数据分析、编辑维护、权限过滤、同步等。

## 数据源总览

| 模块 | 飞书配置 | 说明 |

## 项目资源日历大看板

项目资源日历大看板读取三类飞书表：人员表、资源分配表、人员可用性表。

当前系统部门打通逻辑：

- 优先使用系统“部门管理 / 部门成员”的部门关系。
- 匹配规则为 `飞书人员资源表.姓名 = 系统用户.姓名`。
- 如果匹配成功且系统成员已有部门，大看板中的人员部门使用系统部门路径。
- 如果未匹配、系统未分配部门、或系统存在同名成员导致无法唯一匹配，则回退使用飞书原 `部门` 字段。

## 资源维护台

资源维护台使用的飞书字段与“项目资源日历大看板”的三张资源表一致。

资源维护台额外展示一组系统计算出来的部门同步状态，这些不是飞书字段：

| 状态 | 来源 | 含义 |
| --- | --- | --- |
| 未匹配成员 | 系统计算 | 飞书人员找不到对应系统成员 |
| 已一致 | 系统计算 | 系统部门和飞书部门一致 |
| 系统覆盖待同步 | 系统计算 | 系统部门存在且与飞书不同，可一键写回飞书 |
| 系统未分配 | 系统计算 | 找到系统成员，但系统里还没有分配部门 |

资源维护台同步行为：

- “一键同步系统部门到飞书”只更新已匹配且系统有部门的飞书人员记录。
- 系统部门会覆盖飞书人员表 `部门` 字段。
- 未匹配成员不自动创建飞书人员。
- 系统未分配部门的成员不会覆盖飞书部门。
- “从飞书补齐系统空部门”只补齐系统成员当前未分配部门的情况，不覆盖已有系统部门。

## 其它说明

- 总览页中的其它统计模块多数是系统聚合结果，不一定直接对应某一张飞书多维表。
- “延期预警”主要依赖飞书记录列表里的 `状态`、`截止时间`、`进度`、`风险等级`、`是否阻塞`、`所属项目` 等字段。
- “全局检索”会读取系统内聚合数据和飞书记录列表数据，不是单独的飞书表结构。
- 如果多个项目共用同一张飞书多维表，项目配置中的 `feishuViewId` 用于按视图隔离读取范围。
