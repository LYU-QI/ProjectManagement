# ProjectLVQI 使用手册

## 1. 文档目的

本文档面向项目经理、项目成员、组织管理员与平台管理员，说明当前代码版本下 `ProjectLVQI` 的主要功能、典型操作流程、页面入口与常见问题。

本文档基于当前仓库代码整理，覆盖以下模块：

- 登录与组织/项目切换
- 项目、需求、任务、进度、里程碑
- 成本、工时、风险、效能
- 飞书集成、飞书成员映射
- AI 分析、PM 助手、AI 智能填报
- 自动化规则、任务中心、能力模板
- 知识库、测试管理、Bug 管理
- 组织管理、项目授权、审计日志、套餐设置

## 2. 系统概览

### 2.1 技术形态

- 前端：React + Vite
- 后端：NestJS + Prisma
- 数据库：PostgreSQL
- 外部协作：飞书多维表格、飞书群消息
- 智能能力：AI 周报、PM 助手、智能填报、自动化规则、能力模板

### 2.2 核心使用逻辑

系统围绕两个核心上下文运转：

1. 当前组织
2. 当前项目

很多模块都依赖顶部的组织/项目选择器。

你在页面里看到的数据，通常受以下三层影响：

- 当前组织
- 当前项目
- 页面内的局部筛选条件

尤其是 `飞书集成`、`成本与工时`、`PM 助手`、`任务中心`，都默认跟随当前项目上下文。

## 3. 启动与登录

### 3.1 本地启动

1. 安装依赖

```bash
npm install
npm install --workspace backend
npm install --workspace frontend
```

2. 启动基础服务

```bash
docker compose up -d
```

3. 初始化后端

```bash
cp backend/.env.example backend/.env
npm run -w backend prisma:generate
npm run -w backend prisma:migrate
npm run -w backend prisma:seed
```

4. 启动后端

```bash
npm run dev:backend
```

5. 启动前端

```bash
npm run dev:frontend
```

### 3.2 默认账号

当前仓库说明中的默认测试账号：

- `pm / 123456`
- `lead / 123456`
- `viewer / 123456`

如本地数据库已被修改，请以实际种子数据或数据库内容为准。

## 4. 角色与权限

### 4.1 平台角色

当前代码中常见平台角色包括：

- `super_admin`
- `project_director`
- `project_manager`
- `viewer`

### 4.2 组织角色

系统还使用组织级角色控制管理能力，常见值包括：

- `owner`
- `admin`
- `member`

### 4.3 常见权限差异

- `viewer`：大部分新增、编辑、删除能力会被禁用
- `owner/admin`：可进入组织管理与部分后台管理页面
- `super_admin/project_manager`：拥有更高范围的后台访问能力

### 4.4 权限表现

权限不足时通常会出现以下表现：

- 页面按钮不可用
- 页面顶部提示“当前角色为只读”
- 调用接口时报 `403 FORBIDDEN`

## 7. 典型使用流程

### 7.1 新建项目并开始跟踪

1. 进入 `总览`
2. 新建项目
3. 在顶部切换到该项目
4. 进入 `项目与需求` 录入需求
5. 进入 `进度计划` 录入任务与里程碑
6. 进入 `成本与工时` 录入预算执行与工时
7. 如需飞书协同，进入 `飞书集成` 绑定并同步项目表

### 7.2 做一份项目周报

1. 切换到目标项目
2. 进入 `AI 分析`
3. 生成周报
4. 如需调整周报口径，进入 `能力模板`
5. 编辑 `AI 周报模板`

### 7.3 启用 PM 助手

1. 切换到目标项目
2. 进入 `PM 助手`
3. 配置任务开关
4. 配置定时任务与时区
5. 调整 AI 提示词
6. 先用 `dry run` 手动演练
7. 再切正式发送

### 7.4 用任务中心看系统健康

1. 进入 `任务中心`
2. 查看总量、成功率、失败项
3. 按来源筛选 `PM 助手 / 自动化 / 飞书 / AI 对话`
4. 展开详情查看上下文
5. 对支持的失败项执行重试

## 8. 各模块详细说明

## 8.1 总览

页面用途：

- 查看项目概览
- 管理项目列表
- 快速建立项目基础数据

常见操作：

- 新增项目
- 批量勾选项目
- 批量删除项目
- 行内编辑项目信息

注意：

- 删除项目通常会联动删除该项目下的需求、成本、任务等数据

## 8.2 项目与需求

页面用途：

- 管理需求池
- 跟踪需求状态与优先级
- 做需求评审
- 查看需求变更记录

当前支持能力：

- 新增需求
- 行内编辑需求
- 删除需求
- 批量选择需求
- 按关键字、优先级、状态、评审结果筛选
- 需求评审：通过 / 驳回
- 查看需求变更记录

使用建议：

- 先在顶部切到项目，再录入需求
- “通过/驳回”筛选看的是最近一次评审结果，不是简单状态字段

## 8.3 Todo / 问题池

页面用途：

- 管理工作项、问题、待办
- 适合项目执行阶段持续维护

当前页面传入了：

- 项目列表
- 用户列表
- 飞书成员姓名列表
- 当前项目 ID

因此它本质是一个“项目上下文下的工作项池”。

## 8.4 进度计划

页面用途：

- 管理任务与里程碑
- 查看依赖关系
- 生成风险提示文本

支持能力：

- 新增任务
- 新增里程碑
- 行内编辑
- 删除任务/里程碑
- 管理依赖关系

注意：

- 顶部项目切换会影响这里的上下文
- 里程碑看板与进度计划是互补关系：一个偏表格，一个偏可视化

## 8.5 里程碑看板

## 8.14 飞书集成

页面用途：

- 连接飞书多维表格
- 查询、编辑、导入、导出任务记录
- 行内编辑飞书记录

当前使用逻辑非常重要：

1. 顶部当前项目决定“读取哪张飞书表”
2. 飞书页内的局部筛选只是在当前表数据中再过滤

页面能力：

- 字段配置查看
- 列显示配置
- 提交记录
- 关键字搜索
- 指定搜索字段
- 状态/负责人/风险等筛选
- 分页查询
- CSV 导入导出
- 行内编辑
- 批量删除

注意事项：

- 如果当前已选项目，页面会明确提示“正在读取该项目绑定的飞书表”
- 已选项目时，“所属项目”筛选会被隐藏，避免和项目切换概念混淆

常见问题：

- 能读不能写：通常是飞书应用权限不足或未被加入表协作者
- 列表条数不对：优先检查是否选错了顶部项目

页面用途：

- 生成项目群消息卡片

当前支持任务类型包括：

- 早间播报
- 会议材料准备
- 风险预警
- 超期任务提醒
- 里程碑提醒
- 阻塞任务预警
- 资源负载分析
- 进度看板
- 任务趋势预测
- 周会讨论要点
- 晚间日报
- 周报

当前页面支持：

- 查看任务列表
- 手动运行任务
- dry run 演练
- 指定接收群
- 查看执行日志
- 配置任务开关
- 批量启停
- 配置 cron / timezone
- 项目级提示词配置
- 查看默认提示词

重要规则：

- `PM 助手` 配置支持按项目覆盖
- 生效优先级通常是：
  `项目级模板 > 旧项目级 Prompt > 环境变量 > 默认内置`

使用建议：

1. 先选项目
2. 配置开关
3. 配置时区和定时
4. 配置提示词
5. 先 dry run 再正式发送

## 10. 常见问题

### 10.1 为什么切换项目后数据没有变化？

优先检查：

1. 是否真的切换了顶部项目
2. 当前模块是否还有自己的局部筛选
3. 当前项目本身是否没有数据

### 10.2 为什么飞书列表只有 0 条或 1 条？

优先检查：

1. 顶部当前项目是否正确
2. 是否误把页面内筛选当成了切换数据源
3. 飞书表本身是否绑定在该项目下

### 10.3 为什么飞书能同步读到，但不能修改？

常见原因：

- 飞书应用未加入表协作者
- 应用未开通多维表格写权限
- `FEISHU_APP_ID / FEISHU_APP_SECRET / FEISHU_APP_TOKEN` 不匹配

### 10.4 为什么 PM 助手没有发消息？

优先检查：

1. 是否已配置群聊 Chat ID
2. 是否启用了对应任务
3. 是否使用了 dry run
4. AI 接口是否已配置
5. 飞书发送权限是否正常

### 10.5 为什么任务中心里看不到某类任务？

优先检查：

1. 当前项目是否正确
2. 来源筛选是否限制了显示
3. 该类动作是否真的产生过日志/审计记录
