# 权限控制详细开发计划

## 里程碑 M1：数据模型与基础能力（已完成）
- [x] Prisma 扩展全局角色枚举与项目成员关系模型。
- [x] 新增 `AccessService`，统一计算可访问项目范围。
- [x] 各业务模块（项目/需求/成本/进度/工时/通知/审计/AI）接入访问范围过滤与写入校验。
- [x] ReAct 工具调用接入访问范围与模糊匹配。

## 里程碑 M2：管理平台逻辑分离（已完成）
- [x] 前端增加用户平台/管理平台切换。
- [x] 管理平台导航隔离（审计、系统设置、项目授权）。
- [x] 管理后台页面支持项目成员授权维护。
- [x] 新增用户角色管理接口与前端入口（管理后台可直接改角色）。

## 里程碑 M3：质量与安全加固（已完成）
- [x] 在本地/测试环境完成 Prisma 正式迁移执行并回填历史数据。
- [x] 增加高风险操作审计字段（旧值/新值/操作者/来源）。
- [x] 为角色变更、授权变更补自动化测试。
- [x] 增加越权回归用例（跨项目读写、配置敏感值 reveal、项目授权变更）。
- [x] 增加 Chatbot 无可访问项目时的失败审计回归。

## 里程碑 M4：多人协作上线准备（待开始）
- [ ] 按项目总监维度增加“可管理项目经理名单”策略（更细颗粒）。
- [ ] 增加管理平台首页（权限概览、异常操作告警、授权统计）。
- [x] 产出运维手册：角色初始化、授权流程、审计排查 SOP。
- [ ] 制定物理分离路线（用户端/管理端独立部署）的技术方案。

## 当前阶段 3 收口结论（2026-04-12）

已完成：

- 审计日志新增 `source / beforeSnapshot / afterSnapshot` 字段。
- 高风险管理动作已记录显式审计上下文：
  - 用户创建、改角色、重置密码、删除
  - 组织创建、更新、删除
  - 组织成员邀请、改角色、移除
  - 项目成员授权创建/改角色、移除
- 审计导出 CSV 已包含来源、变更前、变更后字段。
- 高风险管理动作前端已补二次确认：
  - 用户角色调整
  - 用户密码重置
  - 项目授权创建 / 删除
  - 系统配置保存
- 权限回归已覆盖：
  - 跨项目读写拒绝
  - 非项目所有者不得变更项目成员
  - 非超级管理员不得 reveal 敏感配置
  - Chatbot 无可访问项目时失败审计
- 本地数据库已补齐：
  - `AuditLog` 新字段迁移
  - `UserRole.member` 枚举值

阶段 3 已完成。

剩余未纳入阶段 3 的后续项：

- 更细颗粒的管理权限策略
- 管理后台首页治理看板
- 用户端 / 管理端物理分离方案

## 发布步骤建议
1. 执行 Prisma 迁移并备份数据库。
2. 导入初始化授权关系（至少为每个项目补 owner 的 membership）。
3. 先灰度给一位项目总监和两位项目经理验证一周。
4. 观察审计日志和越权告警，再全量放开。
