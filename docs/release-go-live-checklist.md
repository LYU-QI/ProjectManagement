# ProjectLVQI 上线 Checklist

本文档用于阶段 4 的发布准备。执行原则是“可勾选、可追责、可回滚”。

## 1. 环境准备

- [ ] Docker 基础组件正常：PostgreSQL、Redis 已启动
- [ ] 目标环境 `.env` 已配置并完成复核
- [ ] `DATABASE_URL` 指向正确实例
- [ ] 飞书配置与当前租户一致
- [ ] AI 相关配置、Webhook、API Key 已校验

## 2. 代码与构建

- [ ] 已确认本次上线分支、提交号、版本号
- [ ] `npm install` 已完成
- [ ] `npm run -w backend build` 成功
- [ ] `npm run -w frontend build` 成功
- [ ] 如涉及桌面端，`npm run -w desktop dist` 已验证

## 3. 数据库与配置变更

- [ ] Prisma Client 已重新生成
- [ ] 数据库迁移已在目标环境执行
- [ ] 种子数据策略已确认
- [ ] 新增配置项已完成默认值与权限验证
- [ ] 敏感信息展示仍为脱敏状态

## 4. 关键链路回归

- [ ] 登录成功，组织切换正常
- [ ] 项目切换后，各页面数据随项目正确切换
- [ ] 项目与需求：创建、审核、变更记录正常
- [ ] 成本与工时：录入、筛选、统计口径一致
- [ ] 进度计划：任务列表、甘特图、时间线正常
- [ ] 飞书集成：读取、写回、刷新、错误提示正常
- [ ] PM 助手：手动执行、日志查看、失败恢复正常
- [ ] 自动化规则：创建、启停、执行日志正常
- [ ] 任务中心：失败项可定位、可跳转、可重试
- [ ] 审计日志：关键管理动作可追踪

## 5. 文档与运维

- [ ] 使用手册已更新
- [ ] 发布说明已填写
- [ ] 权限与审计 SOP 已确认适配当前版本
- [ ] 值班人、回滚负责人已明确

## 6. 发布后观察

- [ ] `/health` 正常
- [ ] 前端首页可访问
- [ ] 错误日志无新增高频异常
- [ ] 任务中心失败率未异常升高
- [ ] 飞书 / AI / 自动化外部依赖调用稳定

## 7. 回滚门槛

满足任一条件应立即评估回滚：

- [ ] 登录或首页加载失败
- [ ] 项目切换后核心页面普遍空白或错乱
- [ ] 飞书写回持续失败且影响业务
- [ ] 审计、权限或配置安全出现越权风险
- [ ] 数据库迁移导致关键接口异常
